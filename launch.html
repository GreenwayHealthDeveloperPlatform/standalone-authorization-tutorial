STEP 3: Get Authorization Code

In this first half of the authorization, your app identifies itself to the authorization code endpoint using its client ID and the scopes it desires in order to access clinical data. It also provides a 'state' value that identifies the session context that your app establishes with the authorization server. In response, the authorization server displays a consent screen. Once the user grants consent, the authorization server provides your app with an authorization code.
Press the Get button to see what your app transmits, and then grant your consent on the pop up screen in order to obtain a code.

STEP 4: Acquire Access Token

This is the second half of the authorization process. Your app transmits the code obtained in Step 3 and, in exchange, your app receives a token that it uses to authenticate itself to receive FHIR data.
Press the Post button to get a token, and then if desired, press Show to see the token itself.
